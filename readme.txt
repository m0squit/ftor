Инструмент FTOR (Forecasting of Transient Oil Rate)
Цель - предсказание дебита нефти на основе моделей нестационрной фильтрации жидкости в пористой среде.

Инструмент состоит из 2 расчетных библиотек:
1. Библиотека для предсказания количественного притока жидкости к скважине (q_liq)
2. Библиотека для предсказания значений обводненности (wc) для конкретного значения q_liq
---------

На данный момент библиотеки разделены (планируется интеграция).
Соответственно, чтобы выполнить полноценный расчет, необходимо:
 - запустить расчет по 1-ой библиотеке и получить output файл с результатами по прогнозу q_liq
 - запустить расчет по 2-ой библиотеке, используя output файл предыдущего пункта как input файл для проноза wc
---------

Текущее состояние кода адаптировано для запуска расчетов по месторождению Отдельное.
Входные данные для всего расчета:
 - данные МЭР
 - данные ШТР
 - коллекториский и PVT свойства пластов
---------

Смотри файл "Пояснения по запуску.txt" в корневой папке данной библиотеки.

ПОЯСНЕНИЯ ПО БИБЛИОТЕКЕ 2 (модель обводненности)

Входные данные для расчета хранятся в корневой директории:
ftor\tests\data\real\nng\otdelnoe

Файл запуска "main.py" (в папке app).

Основыне настройки расчета (лучше не менять):
    forecast_days_number = 90 - прогноз на 90 дней
    ratio_points_month_day = 0.5 - соотношение чисел точек месячных и дневных данных, используемых для адаптации

Входным файлом для запуска данной библиотеки является:
    Файл "model_liq_rates.xlsx" - должен быть получен после расчета по 1-ой библиотеки (название может быть другим)
    Данный файл поместить в директорию: ftor\tests\data\real\nng\otdelnoe

Выходными файлами такого запуска являются:
    prod_.html (файл с номером скважины содержит прогнозные значения дебитов)
    wc_.html (файл с номером скважины содержит прогнозные значения обводненности)
    performance (файл содержит статистику результатов)

Основной файл (ядро всего расчета) по библиотеке 2 - "model.py" (ftor\libs\flood\corey)
